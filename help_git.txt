	Используемые мной программы с gui для git
gitk
git-gui
gitg

В основном, с git работают из командной строки, но еще доступно два инструмента gitk и git-gui, которые должны установиться вместе с git. Если но не установлены, то вы можете это сделать выполнив команды:

    sudo apt-get install gitk
    sudo apt-get install git-gui
Запускаются они из терминала, нужно находиться в папке репозитория:

	1. Создание репозитория.
  Например:
1 Создаём папку test, в этой папке выполняем команду git init имя_репозитория , будет созданна папка с именем репозитория эта папка будет отслеживаться git, также в этой папке создаётся скрытая папка .git в которой git хранит свои папки и файлы.

git add -A добавляет всё в текущей папке

2 Добавляем файт в репозитория,переходим в папку созданную git, создаём в ней файл test.html с базовой разметкой и заголовком <h1>. Выполняем комманду git add имя_файла или . все файлы в папке, в репозиторий добавлен файл test.html.

3 Делаем первый комит, на данный момент это ветка мастер, выполняем команду git commit -m краткое описание комита - заголовок , коммит добавлен. Если неуказывать -m будет запущен текстовый редактор для описания комита.

4 Делаем второй комит вносим какието изменения в test.html, выполняем команду git add имя_файла происходит индэксирование изменений, выполняем комманду git commit -m краткое описание коммита, коммит добавлен. Существует три соостояния:
  1 Worcing directory - файл или файлы находятся в рабочей директории и там с ними чтото происходит.
  2 Index - выполняется комманда add индэксируются изменения в файлах.
  3 Repository - выполняется комманда commit.

status выводит статус репозитория
log выводит историю репозитория


5 Создание ветки, основная ветка это master, branch имя_новой_ветки - создаёт паралельную ветку проэкта, chekout имя_новой_ветки  -  переключается на работу с этой веткой. head указывает на текущую ветку.

6 Объединение ветки head c master, rebase master, rebase не сохраняет историю объединений. merge имя ветки создаёт коммит слияния

7 restore откат изменений к предедущему комиту
